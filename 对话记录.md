## 需求概述

用户正在学习自然语言处理课程，需要一个 Streamlit Web 应用，展示句子的依存句法树和成分句法树。

### 核心功能要求

1. **输入框**：默认文本 "The boy saw the man with the telescope."
2. **左栏/标签页 - 依存句法**：使用 spaCy + displaCy 渲染依存句法树
3. **右栏/标签页 - 成分句法**：使用 NLTK 或 benepar 渲染成分句法树
4. **代码要求**：包含详细中文注释，自动下载语言模型

## 开发过程

### 1. 初始代码编写

创建了 `app.py`，包含：
- spaCy + displaCy 依存句法可视化
- benepar/NLTK 成分句法可视化
- 自动下载模型逻辑

### 2. 依赖安装

- 安装了 streamlit, spacy, benepar, nltk
- spacy 模型 en_core_web_sm 因网络问题无法下载

### 3. 第一次修改：移除 spaCy 依赖

由于网络无法访问 GitHub，修改代码使用纯 NLTK：
- 依存句法：基于词性标注生成简化的 SVG 依存关系图
- 成分句法：使用 NLTK 的 CFG 解析器

### 4. 第二次修改：移除 benepar

用户要求放弃 benepar，改用 NLTK 自带的 CFG 功能手写三条规则：

```python
规则1: S  -> NP VP   （句子由名词短语 + 动词短语组成）
规则2: NP -> DT NN   （名词短语由限定词 + 名词组成）
规则3: VP -> V NP    （动词短语由动词 + 名词短语组成）
```

### 5. 第三次修改：使用 st.tabs

将左右两栏布局改为 Streamlit 的 `st.tabs`：
- 标签1：📊 依存关系
- 标签2：📚 成分结构

### 6. 第四次修改：修复 SVG 显示问题

**问题1：依存句法树弧线重叠**

解决方案：
- 增加 SVG 高度
- 使用分层弧线，每条弧线错开不同层级显示
- 添加弧线层级计算算法

**问题2：成分句法树叶子节点不对齐**

解决方案：
- 重新设计布局算法
- 所有叶子节点在同一水平线上对齐
- 修复嵌套函数中变量作用域问题

### 7. 第五次修改：体现短语嵌套结构

**问题**：成分树 VP 直接展开为扁平结构，with the telescope 没有体现嵌套

**修改**：重写 `build_heuristic_tree` 函数，使用递归解析实现嵌套结构：

```
S
├── NP (The boy)
│   ├── DT: The
│   └── NN: boy
└── VP (saw the man with the telescope)
    ├── VB: saw
    ├── NP (the man)
    │   ├── DT: the
    │   └── NN: man
    └── PP (with the telescope)
        ├── IN: with
        └── NP (the telescope)
            ├── DT: the
            └── NN: telescope
```

### 8. 第六次修改：添加核心论元提取器

在页面底部新增功能区，展示核心论元：

**提取的论元类型：**
- **ROOT/predicate**: 谓词/根节点
- **nsubj**: 主语
- **dobj**: 直接宾语
- **pobj**: 介词宾语

**展示内容：**
1. 核心论元 DataFrame 表格
2. Markdown 论元结构表
3. 谓词-论元结构可视化
